# AskUserQuestion 交互协议（单源）

> 所有 forge-* skill 的提问格式以本文件为准。各 SKILL.md 只写一行引用，不再复制本内容。

## 提问格式（每次提问遵循）

1. **重新聚焦**：说明当前项目、分支和正在处理的内容。假设用户已经 20 分钟没看窗口。（1-2 句）
2. **通俗解释**：用高中生能听懂的语言解释问题。说清楚"做什么"，不是"叫什么"；不用函数名、不用内部术语。如果你需要读源码才能理解自己的解释，说明解释太复杂了。
3. **给出建议**：`推荐：选择[X]，因为[一句话原因]`。可标注每个选项的 `完整度：X/10`。
4. **列出选项**：`A) ... B) ... C) ...`。涉及工作量时注明估算（如 `人工：~X / AI协助：~Y`）。

按需附加：
- **可视化辅助**：选项涉及 ≥3 个方案对比或复杂流程差异时，用 Mermaid / show-widget 渲染对比图辅助决策；简单 A/B 选择不画图。

## 批量策略（全体系统一）

- 待确认问题 **≤3 个**：逐个单独提问（一次一个，等回答再问下一个）。
- 待确认问题 **>3 个同类**（如审查发现的一批问题）：整理后**批量放入一次 AskUserQuestion**，用编号列出，让用户一次表态；用户要求逐个讨论时再展开。
- 无论单独还是批量：每个选项一句话，5 秒能做决定；问题编号用数字，选项用字母。
