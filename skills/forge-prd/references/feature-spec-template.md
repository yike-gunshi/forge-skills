# Feature Spec 模板

> 作为 PRD 的一部分产出，插入到 PRD 对应功能章节之后。
> 以下 `{变量}` 由 forge-prd 运行时填充。

---

```markdown
## Feature Spec：{功能名称}

**版本**: v{X.Y}
**状态**: [草案 / 已确认 / 开发中 / 已验收]
**关联 Feature**: {feature-id}

---

### 一、用户流程总览

> 从全局视角描述用户在这个功能中如何流转。先看整体，再看局部。

**入口**: {用户从哪里进入这个功能}
**出口**: {用户完成操作后到哪里}

```
{用户} → {步骤1: 动作} → {步骤2: 动作} → ... → {结果}
          │                 │
          ↓ (异常)          ↓ (异常)
        {错误处理}        {错误处理}
```

**流程说明**:
1. {步骤1}: {触发条件} → {预期结果}
2. {步骤2}: {触发条件} → {预期结果}
3. ...

---

### 二、页面/系统结构（全局 → 组件）

> 从最外层到最内层，层层递进。让审阅者先 get 到整体结构，再理解每个部分。

#### 整体布局

```
┌─────────────────────────────────────┐
│              {Header}                │
├──────────┬──────────────────────────┤
│          │                          │
│ {Sidebar}│      {Main Content}      │
│          │                          │
├──────────┴──────────────────────────┤
│              {Footer}                │
└─────────────────────────────────────┘
```

#### 区块职责

| 区块 | 职责 | 包含组件 |
|------|------|---------|
| {区块名} | {一句话职责} | {组件列表} |

#### 组件清单

**{组件名称}**
- 职责: {做什么}
- 交互: {用户怎么操作，操作后发生什么}
- 数据: {展示什么数据，数据来源}
- CSS 约束: {关键视觉属性 — 字号、颜色、间距、圆角等}

> 后端项目替换为：API 拓扑 → 模块职责 → 数据流

---

### 三、行为场景（Given/When/Then）

> 每个功能点 3 个场景：正常 / 异常 / 边界。
> 使用 SHALL 表示强制要求。

#### Requirement: {功能点名称}

{功能点 SHALL 做什么，一句话描述}

##### Scenario: 正常流程 — {场景名}
- **GIVEN**: {前置条件，具体的系统状态}
- **WHEN**: {用户操作，具体到点击哪个按钮/输入什么内容}
- **THEN**: {预期结果，包含：}
  - UI 状态: {页面变化，元素出现/消失}
  - 数据变化: {数据库/状态/API 的变化}
  - 视觉表现: {CSS 属性值 — 如 color: #1e293b, font-size: 14px}

##### Scenario: 异常处理 — {场景名}
- **GIVEN**: {异常前置条件}
- **WHEN**: {触发异常的操作}
- **THEN**: {错误处理方式，包含：}
  - 错误提示: {提示文案和位置}
  - 恢复路径: {用户如何回到正常流程}

##### Scenario: 边界条件 — {场景名}
- **GIVEN**: {边界前置条件 — 空数据/超长输入/并发等}
- **WHEN**: {触发边界的操作}
- **THEN**: {边界处理方式}

---

### 四、验收检查表

> 从上方行为场景自动提取。每条对应一个可验证的断言。
> QA 直接以此为测试用例。

#### 功能验证

- [ ] {Requirement 名}: 正常 — {场景描述} → 预期: {具体可验证的结果}
- [ ] {Requirement 名}: 异常 — {场景描述} → 预期: {具体可验证的结果}
- [ ] {Requirement 名}: 边界 — {场景描述} → 预期: {具体可验证的结果}

#### 视觉/设计合规（对标 DESIGN.md）

- [ ] {组件名} 背景色 SHALL 为 {color value}
- [ ] {组件名} 字号 SHALL 为 {font-size value}
- [ ] {组件名} 内间距 SHALL 为 {padding value}
- [ ] {组件名} 圆角 SHALL 为 {border-radius value}
- [ ] 整体布局 SHALL 符合第二节的结构图

#### 流程完整性

- [ ] 用户流程从入口到出口 SHALL 无断点
- [ ] 所有异常路径 SHALL 有对应的错误提示
- [ ] 所有边界条件 SHALL 有合理的降级处理
```
