# D3 端到端验收报告

> 优化项目的最终计量。验收标准（来自 TODO/goal）：新项目从零跑通一个小需求，数几步、烧多少上下文。
> 所有数字为 2026-07-04 最新 main（含独立评审后续修复）实测。

## 一、接入成本（新项目从零到能开工）

在全新空目录（`git init` 后什么都没有）跑一次脚手架：

| 指标 | 实测 |
|---|---|
| 命令 | `~/.claude/skills/forge-doc-policy/scripts/init-project.sh` |
| 耗时 | **0.03 秒** |
| 铺好文件 | **6 个**（docs/README、docs/INDEX、.features/_registry、docs/bugfix/backlog、CLAUDE.md、.gitignore） |
| 幂等性 | 二次运行"新建 0 项、跳过 6 项"，绝不覆盖 |
| 接入步数 | **1 步**（一条命令）→ 之后 `/forge` 即可用 |

**结论**：从"任何项目都能用起来"的角度，接入成本已降到几乎为零——一条命令、瞬时、安全。

## 二、三条典型链路的真实上下文账本

### 常驻负担（每个会话都在，含非开发会话）

14 个 skill + 总入口的 description 合计 **5.1 KB**。重构前 bugfix/qa 的 description 各是一篇小作文，压缩后整体健康。

### 链路 1 · 轻量改动（小工具、快速修改）

`/forge`（轻量车道判定）→ `forge-eng` 轻量模式（不读 references）

| | 字节 |
|---|---|
| /forge 骨架 | 7 KB |
| forge-eng 骨架 | 19 KB |
| **合计** | **26 KB** |

对比重构前：forge-eng 单文件 28 KB 一次性全读 → 现在骨架 19 KB，轻量模式不碰 references。

### 链路 2 · bug 修复全程（最重的链路，也是重构收益最大处）

关键在**分阶段加载**：走到哪个 P 阶段读哪份手册，旧阶段手册读完即可淡出上下文。

| 阶段 | 当时上下文峰值 |
|---|---|
| P0-P2（环境+定范围） | 骨架 + p0p1 + p2 手册 = **27 KB** |
| P3-P5（复现+根因+修复） | 骨架 + p3p5 手册 = **28 KB**（P0-P2 手册可淡出） |
| P6-P7（QA+验收+分流） | 骨架 + p67 手册 + qa 骨架 + ModeB 手册 = **53 KB** ← 全程最高峰 |

**对比重构前**：bugfix 63 KB + qa 64 KB = **127 KB 一次性全灌**，全程无法卸载。
- 峰值降幅：127 KB → 53 KB（**降 58%**）
- 低谷阶段（P0-P5）只需 27-28 KB，是重构前的约 1/5

这正是骨架化的真实价值：**削峰 + 让旧阶段可淡出**，直接治疗"长会话中途失忆"（复盘里"重新加载 forge-bugfix"那个事故的根因）。

### 链路 3 · 完整功能交付（forge-dev --full）

forge-dev 主上下文 = 骨架 **19 KB**。子 skill（design/eng/qa/review/ship/doc-release）各自在**独立子代理上下文**执行，不叠加到主编排器——这是 forge-dev"上下文工程"设计的红利。主编排器全程只承担自己的 19 KB。

## 三、一个必须诚实说明的反直觉发现

**骨架化没有让"总内容"变少，反而变多了。**

| | 重构前 | 重构后 |
|---|---|---|
| 14 骨架 + 总入口 | 386 KB（就是全部） | **181 KB** |
| 含所有 references 的完整体系 | 386 KB | **974 KB** |

骨架总量降了 53%，但**完整体系翻了 2.5 倍**——因为拆分本身产生了额外内容（每份 reference 的头部说明、按需加载指针、导航表，以及 fupan v2 新增的账本 schema 等）。

**这不是浪费，是 progressive disclosure（渐进披露）的本质**：用"总盘子变大"换"每次上桌的菜变少"。AI 任一时刻只加载当前需要的那一份，而不是被迫端起整桌。之前汇报里说的"降 78%"指的是链路起步成本，容易让人误以为总量下降——独立评审纠正过这一点，此处如实更正。

## 四、验收结论

| 验收标准 | 达成 |
|---|---|
| 新项目从零接入 | ✅ 1 命令 / 0.03 秒 / 6 文件 |
| 小需求有轻量车道 | ✅ /forge 决策树置顶，26 KB 起步 |
| 长会话不再中途失忆 | ✅ bug 链路峰值 127→53 KB，旧阶段可淡出 |
| 真源不漂移、引用不断链 | ✅ 32 references 全 resolve、16 链接 ok、三点一致 |
| 独立评审 | ✅ B- → B+ → A-（三遗留项已关闭） |

**整体判断**：优化项目达成全部验收标准。体系从"能用但笨重"变为"任何项目低成本可用"。
