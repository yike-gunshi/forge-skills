# PRD 变更日志

## [v2026.04.25-fupan-workbench-polish] - 2026-04-25

### 变更背景

用户在真实使用 Fupan Workbench 后反馈：首页时间不是中国上海时区语义、已经完成复盘的任务仍然停留在顶部队列、“表达待优化原话”字号太小、历史复盘详情页缺少可点击目录，整体阅读和导航仍有摩擦。

### 需求调整

- 首页任务和历史时间以中国上海时区展示。
- `consumed` task 视为已完成复盘，必须从首页任务队列出队。
- 没有待处理任务时，首页直接展示历史复盘，不展示空任务面板。
- “表达待优化原话”作为关键学习材料，正文不小于 17px，问题/建议说明不小于 14px。
- 历史复盘详情页为 H2/H3 生成目录和锚点，支持点击跳转。
- `forge-fupan` 在最终复盘生成后应标记对应 task 为 `consumed`。
- Markdown 图片支持点击放大查看，方便检查仪表盘、流程图和截图细节。
- 左侧“学到的知识”必须是编号学习地图，展示知识点数量，不再使用大面积蓝色标签堆叠。
- 每篇最终复盘 Markdown 必须在 frontmatter 后增加 `TLDR`，用 5 条以上短分点提炼关键词、表达方法、关键知识、决策、避坑和下次行动。

### 影响范围

- `docs/PRD.md`
- `docs/DESIGN.md`
- `docs/ENGINEERING.md`
- `skills/forge-fupan/SKILL.md`
- `skills/forge-fupan/workbench/`

## [v2026.04.24-fupan-workbench] - 2026-04-24

### 变更背景

用户反馈现有 `forge-fupan` 复盘文档虽然信息充足，但阅读成本高、专业名词密集、Prompt 诊断有批改感，且复盘前没有询问用户想学习哪些知识、学到什么程度。用户进一步提出希望把 `forge-fupan` 做成本地 localhost 页面：在复盘启动后通过前端完成知识点选择、学习深度选择和反馈提交，最终同时保存 Markdown 和网页阅读版本，历史复盘也能按项目浏览。

### 用户原始需求

用户确认的关键需求：

- 数量和深度都希望由用户选择。
- 触发 `forge-fupan` 后自动打开浏览器页面。
- 尽量在前端页面完成交互、查看结果，不要求用户回命令行。
- 学习深度分为三档：了解、表达、复现。
- 前端减少切换，一个首页分区展示当前待确认任务和历史复盘，点击后进入详情页。
- 所有前端设计环节都要用 Image 2 先生成视觉稿给用户确认。
- 历史复盘模块不显示 “My selections” 或旧选择过程，只显示最终学到了什么。
- 可能同时存在多个复盘任务：首页需要有任务队列，但一次只展开一个任务；AI 只轮询自己创建的 task ID。
- 技术路线倾向 React + FastAPI，但要求轻量、本地、只有用户自己看。
- 功能应直接进入 `forge-cookbook`，让所有使用 Forge Skill 的人都能用。

### 设计方案

新增 Feature Spec：**Fupan Workbench 本地交互式复盘学习工作台**。

核心方案：

1. `forge-fupan` 启动后先分析会话并生成学习地图草案。
2. 系统创建本地 task JSON，状态为 `pending_selection`。
3. 启动或复用绑定 `127.0.0.1` 的 FastAPI 服务。
4. 自动打开浏览器到 React 工作台首页。
5. 用户在页面选择知识区、学习深度和补充反馈。
6. 页面提交后 task 变为 `submitted`。
7. AI 轮询 task 状态，检测提交后自动继续调研和生成复盘。
8. 最终保存 Markdown，并在网页中展示历史复盘和详情页。

### 关键决策

| 议题 | 决定 | 原因 |
|---|---|---|
| 是否做独立产品 | 不做，作为 `forge-fupan` 内置 workbench | 降低范围，直接服务 Forge 复盘流程 |
| 是否所有交互都在前端 | 第一版尽量前端闭环，AI 通过轮询 task 状态继续 | 避免用户提交后还要回命令行说“好了” |
| 学习深度 | 使用 `了解 / 表达 / 复现` 三档 | 更贴近用户学习目标，比四档更清楚 |
| 前端设计确认 | 所有前端设计使用 Image 2 生成视觉稿 | 用户先看效果，减少纯文字设计误差 |
| 历史列表字段 | 只显示学到的知识，不显示 My selections | 用户回看时关心学习成果，不关心当时选择过程 |
| 多任务并存 | 任务队列 + 单任务展开，AI 只消费自己的 task | 防止多个复盘表单堆叠，也避免跨会话误消费 |
| 前端页面数 | 首页工作台 + 详情页 | 首页减少切换，详情页保证阅读体验 |
| 技术路线 | FastAPI + React/Vite 预构建静态前端 + JSON/Markdown | 兼顾交互体验和 Skill 分发 |
| 存储 | 不引入数据库 | 第一版只需本地文件，避免迁移和维护成本 |
| 访问范围 | 只绑定 `127.0.0.1` | 只有本机用户可看，避免外网暴露 |
| 超时策略 | 默认等待 30 分钟 | 防止 AI 无限等待 |
| 失败降级 | 服务失败时降级到对话内确认 | 保证复盘流程不被 workbench 阻断 |
