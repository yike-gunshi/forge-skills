## 需求重述

为 `forge-fupan` 增加一个本地 Fupan Workbench：触发复盘后先生成学习地图草案，自动打开 localhost 页面，让用户选择想学习的知识点、学习深度和补充反馈。用户提交后，AI 轮询自己的 task ID，继续调研并生成最终复盘。历史复盘可以在网页中按项目和时间回看。

## 默认工程偏好

### 技术栈

- 后端使用 FastAPI，绑定 `127.0.0.1`。
- 前端使用 React + Vite 开发，运行态优先托管预构建静态文件。
- 存储使用本地 JSON + Markdown，不引入数据库。
- 不要求用户手动启动 npm dev server。

### 运行方式

- `forge-fupan` 启动时创建 task JSON。
- launcher 尝试复用已有 workbench 服务。
- 若服务不可用，自动选择可用端口启动新服务。
- 启动成功后自动打开浏览器；失败时在对话中输出 URL。
- AI 轮询自己创建的 task ID，默认最多等待 30 分钟。

### 多任务策略

- 允许多个 task 并存。
- 首页显示任务队列。
- 一次只展开一个完整任务表单。
- 当前会话创建的 task 默认置顶并展开。
- AI 不读取、不消费其他会话创建的 task。

### 降级策略

- FastAPI 或依赖缺失时，降级到对话内确认。
- task JSON 损坏时，页面显示可理解错误，AI 可降级到对话内确认。
- 用户超时未提交时，AI 停止等待，页面仍可保留任务供稍后处理。

### 前端体验约束

- 遵循 `docs/DESIGN.md` 的 Local Knowledge Desk 风格。
- 历史列表只展示“学到的知识”，不展示 `My selections`、选择深度或选择过程。
- 所有设计实现以 `docs/DESIGN.md` 为真相源，Image 2 视觉稿作为观感参考。
- QA 阶段用真实截图、CSS 断言和 Feature Spec 验收，不用 Image 2 当通过证据。

## 暂定实现边界

第一版只做：

1. task 创建、读取、提交、轮询。
2. 首页任务队列和当前任务选择表单。
3. 历史 Markdown 扫描和详情页渲染。
4. `forge-fupan` 流程文档改造，说明何时启动 Workbench、何时降级。
5. README / skills reference / CHANGELOG 的最小同步。

第一版不做：

1. 用户账号。
2. 云同步。
3. 数据库。
4. 全文搜索。
5. 知识图谱。
6. 页面内编辑 Markdown。
7. 自动唤醒已结束的 AI 会话。
