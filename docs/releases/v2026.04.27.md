# v2026.04.27 — Fupan 复盘报告结构优化

本次发布把 `forge-fupan` 从“长篇会话总结 + 图表概览”调整为“表达优化、行为诊断、知识学习、AI 协作改进”的复盘工具，并补齐 Codex Desktop 会话读取、压缩后复盘、Workbench 反馈消费和知识模块学习图能力。

## 核心更新

### 报告结构收敛

- 删除“仪表盘速览”和“会话脉络”，不再强制生成 show-widget 仪表盘或 token 精力分布图。
- “工作叙事”改为 2-5 个短段落，只保留目标、关键转折、交付结果和协作问题。
- 删除独立“决策复盘”“避坑指南”“效率词典”“下次行动清单”，相关内容并入行为诊断、速查手册和末尾 TLDR。
- TLDR 移到全文最后，只沉淀需要记住的知识、表达优化和以后操作 SOP。

### 表达与行为复盘重构

- “逐条 Prompt 分析”只保留“表达问题”和“两档更好的说法”。
- “表达问题”要求短、准、具体，避免“表达不够清楚”这类空话。
- “更好的说法”拆成“低关键词版”和“进阶版”：前者不假设用户知道术语，后者引入知识点关键词，帮助形成更像工程任务单的表达。
- “模式诊断”改为“行为诊断”，用“问题 / 分析 / 建议”分析跨 Prompt 的低效协作行为。

### 知识拓展升级

- “知识拓展”从大领域调研改为按单个知识点展开。
- 每个知识点固定包含“是什么 / 怎么用 / 延伸学习”。
- 每个知识点必须先 web research，`延伸学习` 至少提供 1 个可打开的权威来源链接；来源优先官方文档或主流项目文档，不强求每类资源都齐全。
- 每个最终展开的知识模块默认强尝试生成 1 张 Image 2 学习图，放在知识模块标题下方；图片失败时保存 prompt pack、标注待生成，不阻塞复盘正文。

### Claude / Codex 会话兼容

- 预检脚本新增 Codex Desktop 会话定位：优先用 `CODEX_THREAD_ID` 匹配 `~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl`，并兼容 `~/.codex/archived_sessions`。
- 保留 Claude Code `~/.claude/projects/**/*.jsonl` 支持。
- `parse_tokens.py` 同时支持 Claude `message.usage` 和 Codex `event_msg.token_count.payload.info.total_token_usage`。
- Codex token 统计使用累计快照差分，避免流式刷新重复计数。

### 压缩后复盘策略

- 前置检查会统计原始 user/agent/tool/compacted 事件，并输出 `FULL_LOG_AVAILABLE`、`PARTIAL_LOG_AVAILABLE` 或 `SUMMARY_ONLY_OR_MISSING`。
- 如果当前上下文已压缩但本地 JSONL 仍有原始消息，允许从 JSONL 恢复完整复盘。
- 如果只能拿到摘要，降级为轻量复盘：跳过逐条 Prompt 分析，不伪造用户原话、轮次或行为序列证据。

### Workbench 反馈入链

- Workbench 提交的 `selection.feedback` 明确成为全局写作约束。
- 用户补充反馈会影响知识调研、行为诊断取舍、AI 表现复盘和 TLDR/SOP。
- 如果用户反馈与推荐深度或 AI 推断冲突，优先遵循用户反馈；无法执行时必须在最终文档说明冲突。

### AI 表现复盘调整

- 删除固定“需求理解准确度”等四维评分。
- 改为让 AI 基于会话证据自行识别表现问题，并说明用户下次可以提供哪些信息、约束或检查点来帮助 AI 更快做对。
