# 设计变更日志

## [v2026.04.25-fupan-workbench-polish] - 2026-04-25

### 变更背景

真实页面截图暴露出四个体验问题：顶部任务区把已完成内容继续当待办展示、时间语义容易误读、“表达待优化原话”字号偏小、详情页左侧区域没有承担目录导航职责。

二次反馈补充两个阅读问题：Markdown 中的仪表盘/截图需要能放大查看；左侧“学到的知识”大面积蓝色标签过重，既不像地图，也不利于扫读。

三次反馈补充输出内容问题：每篇复盘最上方需要 TLDR，把值得记住的关键词、表达方法、知识和行动浓缩成快速学习入口。

### forge-design L2 轻量审查结论

- **视觉层级:** B → B+。隐藏空任务面板和已完成任务后，首页首屏焦点更清楚。
- **排版:** B- → B+。“表达待优化原话”从 15px 提升到阅读型字号，问题/建议说明也不再像注脚。
- **间距与布局:** B。详情页侧栏从单一知识标签改为目录/标签/源文件三段，阅读路径更符合文档产品心智。
- **交互状态:** B。目录点击跳转补上历史复盘浏览的关键交互。
- **AI 模板痕迹:** A-。继续保持克制的本地工具风格，不新增装饰性卡片或渐变。

### 设计方案

- 首页顶部显示上海时间语义的最近刷新时间。
- 首页任务队列只展示 `pending_selection` / `submitted` / `failed`，`consumed` 进入历史复盘。
- 无待处理任务时不显示空任务面板，历史复盘直接上移。
- 详情页左侧 sticky 侧栏顺序改为：本篇目录 → 学到的知识 → 源文件。
- H2/H3 目录项可点击，H3 缩进显示。
- Markdown 图片点击后进入 lightbox 放大查看，支持 Escape 关闭。
- “学到的知识”改成编号学习地图，显示数量和稳定行高，不再使用满宽蓝色标签堆叠。
- 正文阅读顺序增加 TLDR 第一屏，位于 frontmatter 后、目录前，服务快速学习和回看。

### 设计评分

- 变更前：B
- 变更后：B+
- 门控：L2 通过，可进入工程实现。

## [v2026.04.24-fupan-workbench] - 2026-04-24

### 变更背景

`forge-fupan` 现有输出以 Markdown 长文为主，信息密度高但学习入口困难。用户希望把复盘变成本地 localhost 学习工作台：先在页面中确认要学的知识点和深度，再生成复盘，并在网页中阅读历史复盘。

### 设计方案

采用 **Local Knowledge Desk** 方向：安静、清楚、可信、低干扰。首页作为本地工作台，优先展示当前待确认复盘任务；历史复盘作为次级列表展示；详情页专注阅读，把学习内容放在工程证据之前。

补充设计流程规则：所有前端设计环节都必须用 Image 2 生成视觉稿，让用户先确认实际观感。

### 关键决策

| 议题 | 决定 | 原因 |
|---|---|---|
| 设计方向 | Local Knowledge Desk | 用户目标是学习和回看，不是监控后台或营销展示 |
| 页面结构 | 首页工作台 + 详情页 | 减少首页切换，同时保证详情阅读空间 |
| 配色 | Slate + Blue + Amber | Slate 保持安静可信，Blue 表示操作，Amber 表示当前待确认任务 |
| 字体 | IBM Plex Sans + JetBrains Mono | IBM Plex 适合阅读和 UI，JetBrains Mono 适合路径和技术关键词 |
| 当前任务区 | 首屏置顶 | 复盘启动后用户第一件事是确认学习地图 |
| 历史复盘 | 紧凑列表，只展示“学到的知识” | 用户回看时关心学习成果，不关心 My selections 或选择过程 |
| 多任务并存 | 任务队列 + 单任务展开 | 多个复盘任务可同时存在，但首页不同时展开多个完整表单 |
| 学习深度 | 三段 segmented control | `了解 / 表达 / 复现` 更直观，避免下拉隐藏选择 |
| 文案原则 | 禁止批改感 | 不使用“知识空白”“浪费轮次”等让用户不舒服的表达 |
| 视觉确认 | 前端设计必须生成 Image 2 视觉稿 | 图片比纯文字更容易暴露布局和气质问题 |

### 影响范围

- 新增设计文档：
  - `docs/DESIGN.md`
  - `docs/DESIGN-CHANGELOG.md`
- 影响未来实现：

  - `skills/forge-fupan/SKILL.md` 中复盘输出结构和学习地图确认流程

### 新增 Token

- Slate 中性色体系。
- Blue 操作/焦点体系。
- Amber 当前任务提示体系。
- IBM Plex Sans / JetBrains Mono 字体体系。
- 6-8px 控件/面板圆角体系。
- Image 2 视觉稿门禁。

### 设计门控结果

- 0-10 交互审查综合：B+
- A-F 总体等级：B+
