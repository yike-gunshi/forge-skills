# forge-bugfix：双层验收清单改造

**日期**：2026-04-19
**来源**：用户反馈 bug 修复无明确封闭边界 + 新发现污染当前修复的痛点讨论

---

## 优化背景

v4.0 已经解决隔离问题（worktree + 单 bug），但验收阶段依然是 AI 隐式判断、用户模糊回复。典型症状：
- AI 跑完 QA 说"修好了"，用户随口"嗯"就进了合并
- 用户手动试用时冒出的反馈，要么被 AI 顺手修（scope 蔓延），要么塞进单个 TODO 文件混在一起
- 每次修复的证据（截图 / 日志 / 测试输出）散落各处

本次讨论的原始记录：[`docs/brainstorm-人工反馈收集-2026-04-19.md`](../../docs/brainstorm-人工反馈收集-2026-04-19.md)。

---

## 方案设计

### 架构

单 bug 修复流程加 3 个节点：

```
原 P5 修复 → P6 QA → P7 合并
改为：
P5 修复 → P5.3 生成验收清单 → P6 调用 forge-qa 回填 → P6.5 用户人工验收
        → P7 按三选一分流 → P7.5 新发现分流到 backlog → P8 沉淀
```

### 关键文档

- **单 bug 验收清单**：`docs/bugfix/reviews/BF-{MMDD}-{N}.md`（单文件，表格化，模板在 `skills/forge-bugfix/templates/review-checklist.md`）
- **bug 任务池**：`docs/bugfix/backlog.md`（单一入口，4 区：🐛 待修 / 💡 新需求 / 🌀 待澄清 / 🗄️ 已处理永久保留，模板在 `skills/forge-bugfix/templates/backlog.md`）

### 三选一最终结论（用户填在清单里）

| 结论 | 含义 | 下一步 |
|---|---|---|
| ✅ Pass | 全部验证项通过 | P7 合并 worktree |
| ❌ Fail | 有未通过项（原 bug 范围内） | 回 P5 继续修（不接受新问题） |
| ⚠️ Pass + 新发现 | 原 bug 修好，但看到别的问题 | P7 合并 + P7.5 分流新发现 |

### 新发现分流（P7.5）

AI 读用户写的新发现区，逐条分类写进 `backlog.md`：
- **独立 bug** → 🐛 待修区（分配 BF-XX + P0/P1/P2 优先级）
- **新需求** → 💡 新需求区（建议 `/forge-prd` 立项）
- **模糊反馈** → 🌀 待澄清区
- **同根（并入当前修复）** → **AI 必须举证**（同文件的同一函数 / 同一数据字段 / 同一上游依赖），举证不足默认独立 bug

### P2 捞候选

范围推荐时，AI 从两个来源捞 bug 候选：
1. 用户本会话新报告
2. `docs/bugfix/backlog.md` 的 🐛 待修区（按 P0 > P1 > P2 排序）

## 改动清单

### SKILL.md

- **frontmatter** description 重写，明确双层验收 + 新发现分流的铁律
- **流程总览图**加入 P5.3 / P6.5 / P7.5
- **P2 范围推荐**：从 TODO.md 改为 backlog.md；从双来源捞候选
- **P5.2 范围控制**：硬性拒绝顺手重构；同根必须举证
- **新增 P5.3**：生成验收清单（硬性要求）
- **P6 重写**：必须通过 Skill 工具调用 forge-qa Mode B，带前置门禁 / 参数契约 / 异常兜底（7.5 小节）
- **新增 P6.5**：用户人工验收，等 "验收了" 信号
- **P7 重写**：按三选一分流，移除"弃用 worktree"选项
- **新增 P7.5**：新发现分流到 backlog
- **P8 沉淀**：验收清单即历史产出 + backlog 归档永久保留
- **出口建议**：默认推荐新会话或 `/clear`、`/compact`
- **铁律**：36 条全量规则（拆分单次修复纪律 / 调查 / 验收清单 / 双层验收 / 同根举证 / worktree / backlog / 沉淀 8 组）
- **allowed-tools**：新增 `Skill`（用于调用 forge-qa）
- **清理历史术语**：所有 v5.0 / v4.0 / v3.x / 弹道 / TODO.md 引用清理；历史对比迁移到 `docs/forge-bugfix-changelog.md`

### 新增模板

- `skills/forge-bugfix/templates/review-checklist.md`（单 bug 验收清单）
- `skills/forge-bugfix/templates/backlog.md`（4 区任务池）

## 回测

以本次讨论暴露的痛点为基准：

| 痛点 | 新机制是否解决 |
|---|---|
| "修到第 5 个还没封闭" | ✅ 用户最终结论是硬性封闭节点 |
| 新发现污染当前修复 | ✅ P7.5 强制分流 + 同根必须举证 |
| QA 证据散乱 | ✅ 全部回填到单份验收清单 |
| 长会话 scope 蔓延 | ✅ 出口默认新会话 / /clear / /compact |
| 任务池分散 | ✅ backlog.md 单一入口 |
