# forge-prd：异常态强制项（R8）

**日期**: 2026-04-18
**来源**: 54 篇复盘文档交叉分析 + 本次 5 bug 验收

---

## 优化背景

本次验收的 5 个 bug 中有 3 个（Bug1 图片占位、Bug2 X article 无内容、Bug4 旧数据无 itemId）的根因是：Feature Spec 只覆盖了正常流程，缺少 error/empty/stale 状态的设计。

## 方案设计

### CLAUDE.md 新增 R8 规则

Feature Spec 中每个功能点必须包含至少 3 种异常态场景：
- **空态**：数据为空时的 UI 行为
- **错误态**：网络/API 失败时的 UI 行为
- **降级态**：部分数据缺失时的 UI 行为
- **陈旧态**（涉及本地存储时）：旧版数据结构兼容

### prd-template.md 更新

Feature Spec 模板新增 §3.X.3 异常态场景表，强制填写 Given/When/Then。

### forge-prd SKILL.md 更新

完整性原则中新增 R8 说明，缺少异常态的 Feature Spec 视为未完成。
