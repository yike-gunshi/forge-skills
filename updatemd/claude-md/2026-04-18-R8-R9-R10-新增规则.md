# CLAUDE.md：新增 R8/R9/R10 规则

**日期**: 2026-04-18
**来源**: 54 篇复盘文档交叉分析 + 本次 5 bug 验收根因归类

---

## 优化背景

从 54 篇复盘提炼出 5 类结构性 bug 根因，其中 3 类需要在 CLAUDE.md 层面新增全局规则：

1. **异常态未设计**（Bug1/2/4）→ R8
2. **下游依赖未追踪**（Bug3/5）→ R9
3. **环境未验证**（历史 7+ 次 worktree/端口/路由问题）→ R10

## 新增规则

### R8: 异常态必须设计

Feature Spec 中每个功能点必须包含空态/错误态/降级态场景。涉及本地存储的还需包含陈旧态。Then 子句必须明确指定 UI 行为，不使用"适当处理"等模糊描述。

### R9: 改动必须扫描影响面

涉及类型定义、接口签名、枚举值、store shape 的改动，commit 前必须 Grep 全代码库确认所有引用处已适配。特别注意过滤/分页组件的计数逻辑联动。

### R10: 环境改动必须验证

新增路由 → curl 验证。改 proxy → curl 验证。创建 worktree → diff .env。改后端 → 重启 + curl health。

## 回测结果

以本次 5 个 bug 为基准：5/5 均可在开发阶段被 R8/R9/R10 拦截。
