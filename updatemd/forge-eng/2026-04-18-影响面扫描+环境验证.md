# forge-eng：影响面扫描 + 环境验证

**日期**: 2026-04-18
**来源**: 54 篇复盘文档交叉分析（高频 bug 根因归类）

---

## 优化背景

从 54 篇复盘中发现两类高频 bug：
1. **下游依赖未追踪**（8+ 次）：新增/修改字段后，消费该字段的下游组件未同步更新。典型案例：pill 过滤后计数不联动、前端覆盖后端排序、枚举新增未全局搜索。
2. **环境/运行时未验证**（7+ 次）：worktree 缺 .env、Vite proxy 端口错、路由未 curl 验证、进程 CWD 不匹配。

## 方案设计

### 新增步骤 5.6：环境验证（R10 强制项）

Worktree 创建后、编码前，强制通过 4 项检查：
- .env 完整性（diff 主仓库）
- 后端启动（curl health）
- 前端启动（curl localhost）
- 代理配置（curl 通过代理的 API 路径）

### 新增步骤 7.5：影响面扫描（R9 强制项）

每个 Wave 完成后、commit 前：
- 从 git diff 提取所有新增/修改的类型定义、接口签名、枚举值、store shape
- Grep 全代码库找到所有引用处
- 逐一确认是否需要适配
- 输出影响面清单，未处理项必须在下一 Wave 前完成

### 新增步骤 7.8：Post-Implementation 验证（R10 强制项）

所有 Wave 完成后：
- 新增 API 路由 curl 验证
- 前端无 console.error
- 配置变更 diff 确认
